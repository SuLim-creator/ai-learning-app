아래 순서대로 실행하세요.

1. `npx tsc --noEmit 2>&1` 실행
2. 에러가 없으면 "타입 에러 없음"으로 종료
3. 에러가 있으면 각 에러를 분석해 수정
4. 수정 후 다시 타입체크 실행
5. 최대 3회 반복 후 결과 보고

## 수정 규칙

- `as any`, `@ts-ignore`, `@ts-expect-error` 사용 금지
- 올바른 타입 정의나 타입 가드로 해결
- 타입이 불명확하면 `unknown`으로 받아 narrowing 처리
- 외부 라이브러리 타입이 없으면 `@types/패키지명` 설치 제안

## 출력 형식

각 수정 사항을 아래 형식으로 보고:

```
수정 전: 파일경로:라인번호 — (에러 내용)
수정 후: (어떻게 고쳤는지)
```

3회 후에도 에러가 남으면 "자동 수정 불가 — 수동 검토 필요" 항목으로 남은 에러를 나열하세요.
