`git diff`로 스테이징되지 않은 변경사항과 스테이징된 변경사항을 모두 확인한 후, 아래 항목을 순서대로 점검하고 **한국어**로 보고하세요.

## 점검 항목

1. **로직 에러**
   - 잘못된 조건문 (경계값, 부등호 방향)
   - 비동기 처리 누락 (await 빠짐, Promise 미처리)
   - null/undefined 체크 누락

2. **타입 안전성**
   - `any` 타입 사용
   - `as Type` 강제 단언 남용
   - 타입 좁히기(narrowing) 없이 유니온 타입 사용

3. **보안 이슈**
   - 인증/인가 누락된 API 라우트
   - 사용자 입력 미검증 (XSS, SQL injection)
   - 민감 정보 노출 (console.log, 하드코딩된 키)

4. **CLAUDE.md 규칙 위반**
   - `any` 사용 금지 위반
   - `@ts-ignore` 사용
   - 파일명 kebab-case, 컴포넌트명 PascalCase 위반

## 출력 형식

문제가 있으면 파일:라인 형태로 구체적으로 보고하세요.

```
[항목] 파일경로:라인번호
문제: (무엇이 문제인지)
제안: (어떻게 고쳐야 하는지)
```

문제가 없으면 "리뷰 완료 — 발견된 문제 없음"으로 끝내세요.
